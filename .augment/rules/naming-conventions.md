# Naming Conventions

## Purpose

This document defines naming standards for the Tayyibt project.

It ensures:
- Consistent naming across all services
- Readable and maintainable code
- Predictable structure for developers and AI tools

---

## General Rules

- Use clear, descriptive names
- Use English only
- Avoid abbreviations unless widely known (e.g., ID, API)
- Be consistent across the entire codebase
